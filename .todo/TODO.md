- [ ] **[DB/Migración] Plan de reversión + ensayo antes de pushear la migración de roles dinámicos a dev remoto** — La migración de roles dinámicos (tablas `roles`+`role_modules`, funciones `is_admin`/`has_module`, trigger `prevent_rol_self_change`, políticas RLS, y la transformación de datos de `usuarios.rol`) está aplicada y verificada **solo en local**; el dev remoto (`ydcadspinryybextlvyi`) y prod siguen intactos. **Gatea el PR a `development`:** el `supabase db push` a dev debe ir justo ANTES del PR, y antes de ese push hay que tener listo el plan de reversión y ensayarlo.
